## 📋 项目背景

### 业务需求

采集美团平台上医美机构的项目数据，用于市场价格分析。

**目标数据：**
- 医院名称
- 项目名称
- 项目价格
- 销量（如有）

**采集范围：**
- 指定医院列表（如：郑州美莱、郑州天后等）
- 每家医院的所有展示项目

| F1.5 | 服务保活 | 后台持续运行，不被系统杀死 | P1 |

| ID | 功能 | 说明 | 优先级 |

| ID | 功能 | 说明 | 优先级 |

## 🏗️ 技术架构

### 架构选型

```
┌─────────────────────────────────────────────────┐
│                   用户界面层                      │
│  (MainActivity - 配置、状态、数据查看)            │
├─────────────────────────────────────────────────┤
│                   业务逻辑层                      │
│  (CollectorManager - 采集调度、任务管理)          │
├─────────────────────────────────────────────────┤
│                   核心服务层                      │
│  (MedicalBeautyCollectorService - 无障碍服务)     │
│  - AccessibilityEvent 监听                       │
│  - AccessibilityNodeInfo 遍历                    │
│  - Gesture 模拟 (滚动、点击)                      │
├─────────────────────────────────────────────────┤
│                   数据存储层                      │
│  (Room/SQLite - 本地数据、配置、日志)             │
├─────────────────────────────────────────────────┤
│                   网络层                          │
│  (Retrofit - API 通信、数据同步)                  │
└─────────────────────────────────────────────────┘
```

### 技术栈

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 开发语言 | Kotlin | 比 Java 简洁，官方推荐 |
| 最低版本 | Android 8.0 (API 26) | 覆盖主流设备 |
| 无障碍服务 | AccessibilityService | Android 系统服务 |
| 本地数据库 | Room (SQLite) | 结构化数据存储 |
| 网络请求 | Retrofit + OkHttp | HTTP 客户端 |
| 依赖注入 | Hilt | 简化依赖管理 |
| 异步处理 | Coroutines | 协程处理异步任务 |

## 📊 数据模型

### 医院 (Hospital)

```kotlin
data class Hospital(
    val id: Long = 0,
    val name: String,              // 医院名称
    val city: String = "郑州",      // 城市
    val status: HospitalStatus,    // 采集状态
    val createdAt: Long,           // 创建时间
    val updatedAt: Long            // 更新时间
)

enum class HospitalStatus {
    PENDING,      // 待采集
    IN_PROGRESS,  // 采集中
    COMPLETED,    // 已完成
    FAILED        // 失败
}
```

### 项目 (Project)

```kotlin
data class Project(
    val id: Long = 0,
    val hospitalId: Long,          // 所属医院 ID
    val title: String,             // 项目名称
    val price: Double,             // 价格
    val sales: Int = 0,            // 销量
    val rawPrice: String? = null,  // 原始价格文本
    val collectedAt: Long          // 采集时间
)
```

### 任务 (Task)

```kotlin
data class Task(
    val id: String,                // 任务 ID (API 返回)
    val type: TaskType,            // 任务类型
    val hospitalName: String,      // 目标医院
    val status: TaskStatus,        // 任务状态
    val result: TaskResult?        // 采集结果
)

enum class TaskType {
    COLLECT_HOSPITAL,   // 采集医院
    COLLECT_CATEGORY    // 采集品类
}

enum class TaskStatus {
    PENDING,
    RUNNING,
    SUCCESS,
    FAILED
}
```

## ⚠️ 风险与挑战

### 技术风险

| 风险 | 描述 | 应对方案 |
|------|------|---------|
| R1 | 美团反自动化检测 | 模拟人工操作、随机延迟 |
| R2 | UI 结构变化 | 多选择器适配、定期更新 |
| R3 | 无障碍服务被限制 | 保活策略、用户引导 |
| R4 | 数据采集不完整 | 日志记录、失败重试 |

### 合规风险

| 风险 | 描述 | 应对方案 |
|------|------|---------|
| R5 | 违反美团用户协议 | 仅限个人学习研究使用 |
| R6 | 数据使用合规 | 不用于商业竞争、不公开传播 |

## 📝 待讨论事项

### 技术决策

1. **Kotlin vs Java** - 推荐 Kotlin，更简洁
2. **Room vs SQLite** - 推荐 Room，更现代化
3. **是否支持多 App** - 美团/新氧/更美等
4. **是否支持云端同步** - 多设备数据同步

### 产品决策

1. **是否做独立 App** - 还是作为服务嵌入
2. **是否开放配置界面** - 还是硬编码配置
3. **是否支持定时采集** - 还是手动触发
4. **数据导出格式** - CSV/Excel/JSON
